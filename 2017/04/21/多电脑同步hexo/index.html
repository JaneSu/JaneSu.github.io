多电脑同步hexo

发表于 2017-04-21 | 分类于 Git |

考虑到在不同电脑之间写博客，以及日后可能会发生的在新电脑使用博客。所以在网上搜了“如何多电脑同步hexo”。

我的目标是通过Github托管博客的文件，然后在不同的电脑之间同步。假定看这篇博客的人已经安装好HEXO，并且能正常维护博客。有关node.js，Git的安装我在这里不会重复，如果有疑问，可以参考网上的其他方法或是参考我的另一篇博文博客建站过程记录.

说明：

电脑A是最开始安装hexo的电脑

电脑B是后期想要写博客同步的电脑

两台电脑都用的hexo文件夹命名，分别为hexo_A和hexo_b

电脑A

在hexo_A下初始化git

在Github上建立一个新的仓库，我的命名为blog，这里有一个地方需要注意，为了避免后面麻烦，创建仓库的时候不要选择创建README.MD文件，只要有一个空的仓库即可。之所以这样做，是为了避免将远程仓库和本地仓库关联时可能导致的冲突，我的GIT还在入门，暂时也没想出好方法解决这个问题，所以我的建议仅供参考。

关联远程和本地的仓库

推送本地的内容到远程库

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

至此电脑A的操作结束

电脑B

在全局安装hexo(这一步貌似不是必须的)

在你选的文件夹内克隆远程库

这样就会在本地创建一个名为hexo的文件，里面包含远程库的所有内容。

之后分别执行

这里需要注意，不要执行hexo init，也就是不要初始化这个文件夹的hexo，否则会覆盖设定好的hexo配置文件。

建议这里先执行

这么做的原因在于，如果这时候你的博客是有问题的，而你推送到远程库，即便可以回溯，但是添了不必要的麻烦。所以先通过这一步检查在本地有没有问题。比如我的预览就是一片空白，google了一下原因，有两种可能：

  1. themes下的主题文件的名称和配置文件里的不一样。这种只要修改名称一致即可
  2. 以我举例，我安装的是next主题，这个文件下是空的

所以我的解决方法比较直接，从电脑A里把整个next文件拷贝过来，覆盖hexo_B/themes里的next文件，然后删除next里的.git文件，否则后期会报错。

之后就可以执行正常的推送步骤。
