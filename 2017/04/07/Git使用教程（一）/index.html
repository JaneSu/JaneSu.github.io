Git入门（一）：git的介绍和简单使用

发表于 2017-04-07 | 分类于 Git |

本文会讲一些Git的基础知识，以及简单的使用。后续的博客会继续深入下去。我尽量不去重复网上已有的内容，而是写出自己的理解。

什么是Git

Git是一种版本控制系统，和其他大多数版本控制系统相比，Git是分布式的，优点是速度更快，信息保存的更安全。同时，git的使用并不必须借助于网络，离线的情况下依旧可以使用，这得益于初始化git时，会在当前项目的根目录创建一个.git的目录，这里面会保存所有关于git的操作信息。可以将其理解成一个人的档案，档案中会有一个人生平各种信息的记录，通过档案就能了解一个人，而git往往能做的更多。

既然是版本管理，必然会有历史回溯的问题，如何回到自己的某一次提交呢？简单而言，每一次的提交都会产生一个独属于该版本的号码，就像是一个“标签”，就是通过这个“标签”索引你需要的内容。而这个‘标签’是由40个十六进制字符（0-9和a-f）组成的字符串，这个叫SHA-1 散列（哈希）。

git的工作原理很容易理解，但在这之前，要先知道git的各种状态，也就是该步操作处于整个流程的什么阶段。

Git 有三种状态，你的文件可能处于其中之一：已提交（committed）、已修改（modified）和已暂存（staged）。已提交表示数据已经安全的保存在本地数据库中。已修改表示修改了文件，但还没保存到数据库中。已暂存表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。

这里说的是git的三种状态；但同时git也有工作区域之分，分别是Git仓库、工作目录、暂存区域。

他们之间的关系可以用一张图说明：

  • Git 仓库目录是 Git 用来保存项目的元数据和对象数据库的地方。这是 Git 中最重要的部分，从其它计算机克隆仓库时，拷贝的就是这里的数据。
  • 工作目录是对项目的某个版本独立提取出来的内容。这些从 Git 仓库的压缩数据库中提取出来的文件，放在磁盘上供你使用或修改。
  • 暂存区域是一个文件，保存了下次将提交的文件列表信息，一般在 Git 仓库目录中。有时候也被称作`‘索引’’，不过一般说法还是叫暂存区域。

我理解的是，Git仓库目录就是前面说到的.git记录的当前项目的信息，而工作目录是当前进行操作的项目文件夹（.git就存在该文件夹内），而暂存区是对项目文件的一次扫描记录，包含你修改了哪些内容，在下次提交。

对于这里的各种名词不用太纠结，我初次看的时候也是很困惑，但是实际使用过程中就能理解这里说的是什么了。

git的简单使用

git的基础介绍就到此为止，我讲的并不全面，以后我再有深入的理解或是看到新内容需要补充，我会在添加进来。

接下来的内容是关于git的安装和入门的使用，不会有特别难的部分，因为我目前用的也不是太多，接下来的部分会在以后的博文里慢慢说的。

安装git

要说安装之前，先要弄清楚git有几种形式。得益于苹果推出的图形界面，接下来的计算机发展都是按照这个路数来的，不像最早的Dos命令行，初学者不用记住各种各样的命令，直观的简化了操作难度，因此在git的使用方面也分为两派：GUI和命令行。

Git可以在这里进行下载：git下载。

不过我当初采用的是另一种方法，安装GitHub Desktop.这个的优点是同时会安装两套程序，一个是我们常见的图形操作界面，另一种是具有命令行作用的git shell。所以这个完全看使用者，我最初觉得命令行麻烦，并且当时对于git的使用完全是依赖于Github，因此前者完全能满足我的需求，但是后来安装这个博客，同时使用gulp，命令行的重要性就凸显出来了，并且，我现在觉得命令行非常好用，能提供的功能也比GUI多。

那我就按照命令行的使用来介绍吧。

初始化git

通过命令行进入当前的项目文件夹，执行命令：

git init

这样就会在该项目创建.git用来记录所有操作。

添加和提交

这是本文的重点，好在git的常见操作其实不多，难的在于这些操作延伸出去的部分，但是这里并不会对这一部分有所涉及，都是后话。

还记得前面讲到的git的几种状态么？已提交，已修改，已暂存。这里面已修改指的是对当前项目文件的一种修改，不涉及操作，所以主要是另外两种状态。

当你添加，修改，删除完文件之后，执行命令：

git add <fileName>

比如说，当前文件内有一个readme.md文件，则：

git add readme.md

如果是添加css路径下的所有文件，则：

git add /css

如果嫌麻烦，可以通过下面的命令一次添加所有文件：

git add .

这里需要对git add做一个说明，这一命令具有两种效用：

  1. 将新添加的文件或是文件夹加入到暂存区
  2. 将本地修改的内容添加到暂存区

因此不论做什么，git add都是git命令的第一步。

将文件添加到暂存区之后，就是提交：

git commit -m“description”

description是对当前提交版本的一种描述，以备以后回退版本时你知道你做了什么，否则就是一团乱麻。

查看状态

进行了添加或是修改的操作之后，如果想查看当前处于什么状态也是可以做到的，比如说修改了readme.md里的内容，但是没有执行添加操作，执行

git status

会看到以下内容：

可以看到Changes to be committed下面出现的内容，表示当前做的修改还没有被添加到暂存区。这只是一种情况，其他的也很类似，只是表述的语言可能不太一样，所以在这里我就不多说了。

查看差异

除了上述说到的查看文件的状态，还有一种情况是你想知道更具体的，比如说你这两次修改，或是工作目录和已暂存文件之间的差异，则可以执行这个命令：

git diff

如果是你想知道已经暂存的文件将要添加到下次提交里的内容，可以执行：

git diff --staged

这里有一个小坑，可能会有这样的情况，运行了git diff之后却是毫无显示，不要急，这不是问题，而是没弄明白diff这个行为的含义。这个命令的根本是告诉你你已经修改了工作区的文件，和之前暂存的暂存区的文件的差异，如果都已经添加到了暂存区，那就不存在已修改未暂存的文件，自然也就不会有两者之间的差异。
