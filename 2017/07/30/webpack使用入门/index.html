webpack使用入门

发表于 2017-07-30 | 分类于 工具 |

现在流行的打包工具有很多，之前接触过gulp，但是和工作项目中的匹配度太低，因此只能说是尝试过，但是没怎么好好使用。

这次之所以换了方向使用webpack，是最近在使用的vue选择的工具就是它，所以为了加深了解，为了更好的使用vue，因此选择了这个工具。那下面我会记录我开始使用这个工具的方法，和碰到的问题。

一些比较基础的问题，比如安装node.js和npm，这个我就不再写了，网上各种各样的文章一大堆。我的文章也会按照业务流程来写，方便我以后查阅。

安装webpack

安装webpack分为两种，一是在全局安装，一个是在项目文件下安装。

在我看过的的很多教程里都在提倡使用第二种方法，因为这样你不会面临升级的问题，不会出现版本不一致导致无法使用的问题。但在我自己的使用过程中，如果不全局安装，在运行webpack是就会提示command not found webpack。而全局安装就能避免这个问题。

所以实际上先进行全局安装，再在项目问价下安装一遍的方式比较好.

这样，你的项目文件里会有一个package.json的配置文件，一个名为node_modules的文件夹。

loader

关于loader，我理解的不深，因此我也只能粘贴网上的描述

Webpack 本身只能处理 JavaScript 模块，如果要处理其他类型的文件，就需要使用 loader 进行转换。

Loader 可以理解为是模块和资源的转换器，它本身是一个函数，接受源文件作为参数，返回转换的结果。这样，我们就可以通过 require 来加载任何类型的模块或文件，比如 CoffeeScript、 JSX、 LESS 或图片。

所以从这个描述里可以看出的是，loader本质上就是一个转换器，将一种资源转换成js可以识别的内容，然后使用。

loader是webpack重要的组成部分，他有很多特性：

  • Loader 可以通过管道方式链式调用，每个 loader 可以把资源转换成任意格式并传递给下一个 loader ，但是最后一个 loader 必须返回 JavaScript。
  • Loader 可以同步或异步执行。
  • Loader 运行在 node.js 环境中，所以可以做任何可能的事情。
  • Loader 可以接受参数，以此来传递配置项给 loader。
  • Loader 可以通过文件扩展名（或正则表达式）绑定给不同类型的文件。
  • Loader 可以通过 npm 发布和安装。
  • 除了通过 package.json 的 main 指定，通常的模块也可以导出一个 loader 来使用。
  • Loader 可以访问配置。
  • 插件可以让 loader 拥有更多特性。
  • Loader 可以分发出附加的任意文件。

更多loader的使用还是需要google自己查了，我这里只有一个示例仅供参考。
